# 의사결정·전략

이 카테고리는 결정자(임원·이사회·주주·투자위원회·클라이언트 임원)가 한 번 보고 결정·승인·자원 배분에 들어가도록 만드는 모든 산출물을 다룬다. 기획·제안·이사회·IC·임원 PPT·컨설팅 권고 덱·6-pager·스토리라인이 여기 속한다. 공통점은 (1) 결정자의 시간이 비싸다, (2) 첫 페이지·첫 줄에 결정 요청 또는 메시지가 와야 한다, (3) 약속의 인플레이션을 피한다, (4) 액션 타이틀과 So what이 끝까지 이어진다.

## 종결 체 원칙

본문 텍스트와 메모는 `~이다` 체로 통일한다. PPT·덱 계열(임원용 PPT·컨설팅 덱)은 헤드라인과 액션 타이틀을 `~함·~임` 명사형 종결로 따로 쓴다. 슬라이드 한 장의 결론을 압축하는 자리이므로 단정형 종결어미가 가장 짧고 강하다. 한 산출물 안에서 헤드라인 `~함`과 본문 `~이다` 체가 분리되는 것은 명시적 허용이다.

예. 헤드라인 "Q3 ROAS 2.1로 목표 미달함" / 본문 "ROAS 부진의 핵심은 신규 채널 비중이 38%로 과도했다는 점이다."

## 매핑 표

각 행의 RED·SILVER·BLUE는 5-Color Harness 페르소나 정체성에 해당한다. RED 이성(논리·근거·결론 타당성), SILVER 분야 전문가(베테랑이 첫눈에 짚는 디테일), BLUE 공감(결정자가 부담 없이 결정에 들어가는 신뢰 무게). 의사결정·전략 카테고리는 결정자(임원·이사회·주주·IC·클라이언트 임원)가 BLUE의 대상이다. 합격선이 표기되지 않으면 9.5다.

|------|---------------------|------------------------------|----------------------------------|--------|
| 기획안·제안서 | 전략의 인과 정합성 | 기획 베테랑이 보는 논리 흐름 | 결정자가 그 자리에서 다음 단계를 정하게 만드는 구체성 | 9.5 |
| 영업 제안서·RFP | 차별성·적합성의 근거 정합성 | 영업 베테랑이 보는 구조 | 평가위원이 가격·일정 의문 없이 점수를 주는 명확성 | 9.5 |
| 이사회 보고서 | 핵심 의제·결정 요청의 정합성 | 이사회 사무국 베테랑이 보는 논리·데이터 흐름 | 이사가 5분 안에 의결하게 만드는 가독성·위계 무게 | **9.7** |
| 주주 서한 | 메시지의 무게·근거 정합성 | CEO 서한 베테랑이 보는 자신감과 겸손의 톤 균형 | 주주가 끝까지 읽고 신뢰 한 단계 올라가는 분량·가독성 | **9.7** |
| PRD | 사용자 문제·솔루션의 인과 정합성 | 시니어 PM 베테랑이 보는 구조·우선순위 | 개발팀이 구체성·측정가능성 의문 없이 착수하는 명료함 | 9.5 |
| 투자 메모 (IC Memo) | 투자 논리·리스크의 인과 정합성 | 시니어 투자 파트너가 보는 비즈니스·시장·재무·리스크·exit 다섯 단 | IC 위원이 가정 표시·정량 정성 균형 의문 없이 결정에 들어가는 신뢰 | **9.7** |
| 스토리라인 (Pyramid Principle) | 답의 인과 정합성 | 컨설팅 시니어가 보는 MECE 지지 답 구조 | 결정자가 문장의 단정성으로 답에 한 번에 동의하는 무게 | 9.5 |
| 내러티브 메모 (6-pager) | 결론 강도·가설의 정합성 | 메모 작성 베테랑이 보는 MECE 산문 흐름 | 리더가 30분 침묵 독해 후 결론에 첫 질문이 꽂히는 산문 통일 | 9.5 |
| 임원용 PPT | 결론 한 줄·근거·인과 정합성 | 임원 보고 베테랑이 보는 두괄식 구조와 액션 타이틀·전환 | CEO가 5분에 결정 한 줄을 잡게 만드는 위계·여백·강조 무게 | 9.5 |
| 컨설팅 덱 (진단·권고) | So what·인사이트의 정합성·답 일치 | 컨설팅 EM 베테랑이 보는 액션 타이틀 종결과 페이지 흐름 | 클라이언트 CEO·CFO·COO가 30분에 권고 채택을 결정하는 무게 | 9.5 |

## 합격선 기준 장면

- 기획안·제안서. "의사결정자가 한 줄 요약을 보고 다음 검토 회의를 잡고 일정·비용 페이지로 직행한다."
- 영업 제안서·RFP. "구매 담당자가 첫 3페이지에서 점수를 매기고 인터뷰 short-list 후보로 올린다."
- 이사회 보고서 (9.7). "이사회가 첫 페이지에서 결정 요청을 정확히 알아채고 본문은 보충 자료로만 본다."
- 주주 서한 (9.7). "주주가 한 분기 더 가져갈 신뢰를 결말 한 단락에서 확인한다."
- PRD. "엔지니어가 한 번 읽고 user story를 카드로 끊고 P0/P1/P2 우선순위를 그대로 가져간다."
- 투자 메모 (9.7). "IC가 한 번 토의로 결정을 내리고 베이스·업·다운 케이스를 추가 질문 없이 받아들인다."
- 스토리라인. "한 페이지 답와 지지 답 셋만 보고 본 덱 30장의 흐름을 예측한다."
- 내러티브 메모. "리더가 첫 30분 침묵 독해 후 첫 질문이 결론 한 줄에 정확히 꽂힌다."
- 임원용 PPT. "CEO가 슬라이드 10장을 5분 안에 넘겨도 결정 한 줄을 잡는다."
- 컨설팅 덱. "클라이언트 임원이 액션 타이틀만 훑고 30분에 권고 채택을 결정한다."

## 문체·금지 뱅크

### 기획안·제안서
- 종결 체. `~이다` 체.
- 문체. 한 줄 요약 → 배경 → 솔루션 → 실행 계획. 의사결정자 관점.
- 분량. 1500~3000자, A4 2~5페이지.
- 금지. 솔루션 없는 문제 나열. 실행 일정 부재. 비용 추정 부재.

### 영업 제안서·RFP
- 종결 체. `~이다` 체.
- 문체. 첫 페이지에 차별성 한 줄. 가격과 일정을 명확하게. 클라이언트 언어.
- 분량. 5~15페이지. 핵심은 첫 3페이지에.
- 금지. 자사 자랑이 차별성보다 길어짐. 모호한 가격("협의"). 일정에 버퍼 부재.

### 이사회 보고서. 9.7
- 종결 체. `~이다` 체. 격식 단정.
- 문체. 결정 요청 한 줄을 머리에. 자신감과 겸손의 균형. 숫자는 출처와 함께.
- 분량. 안건당 1~2페이지, 1500~3000자. 첨부 자료 별도.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - 자기 변호.
  - "검토 중" 직접 인용 금지.
  - "선방" 직접 인용 금지.
  - 약속의 인플레이션.
  - 결정 요청 누락(보고서로 끝나는 안건).

### 주주 서한. 9.7
- 종결 체. `~이다` 체.
- 문체. 핵심 메시지 한 줄을 머리에. 자신감과 겸손의 균형. 숫자는 출처와 함께.
- 분량. 2~5페이지, 3000~8000자. 분기 서한은 짧게, 연차 서한은 길게.
- 금지 (9.7군).
  - 자기 변호.
  - "견조한" 직접 인용 금지.
  - "전략적 우위" 직접 인용 금지.
  - 약속의 인플레이션.

### PRD
- 종결 체. `~이다` 체.
- 문체. 사용자 문제 한 줄, 솔루션 한 줄, 측정 지표 한 줄. P0/P1/P2 분류.
- 분량. 1500~5000자. 하나의 기능 단위면 1500~2500자, 큰 제품 단위면 3000~5000자.
- 금지. "좋은 UX", "더 편하게" 같은 추상 표현. 비범위(out of scope) 부재. 측정 방법 없는 성공 지표.

### 투자 메모 (IC Memo). 9.7
- 종결 체. `~이다` 체.
- 문체. 투자 결정 한 줄을 머리에. 비즈니스·시장·재무·리스크·exit 다섯 단. 가정 명시. 정량과 정성 균형.
- 분량. 5~15페이지, 7000~20000자. 사전 메모는 1~2페이지.
- 금지 (9.7군).
  - "유망" 직접 인용 금지.
  - "잠재력 있음" 직접 인용 금지.
  - "검증 필요" 직접 인용 금지.
  - exit 시나리오 부재.
  - 베이스·업·다운 케이스 부재.

### 스토리라인 (Pyramid Principle)
- 종결 체. `~이다` 체. 답는 단정 서술형(의문문 아님).
- 문체. 답 한 줄. 지지 답 셋(MECE). 각 지지 답에 사실 두세 개. So what 끝까지.
- 분량. 한 페이지. 본 덱 작성 전 단계 산출물.
- 금지. 답가 질문형. 지지 답 MECE 위반. 사실 없는 추론. 한 답에 이슈 두 개.

### 내러티브 메모 (6-pager)
- 종결 체. `~이다` 체. 본문은 완결된 문단으로(불릿 금지).
- 문체. 결론 한 문장을 첫 단락에. 본문은 결론을 지지하는 문단. 표·차트는 부록 별지로 분리. FAQ가 있으면 본문 뒤에 별도 섹션.
- 분량. 1-pager 800~1500자(영문 약 350~600 단어). 6-pager 4500~6000자(영문 1500~1800 단어). 본문 외 부록·데이터·FAQ는 분량 제한 밖.
- 금지. 본문에 불릿·번호 매김 사용. 본문에 표·차트 삽입(부록으로 분리). 결론 부재한 정보 나열. "검토가 필요"로 끝맺기. 한 단락에 결론 두 개. 회의 직전 배포(침묵 독해 시간 부재).

### 임원용 PPT
- 종결 체. 헤드라인·액션 타이틀 `~함·~임` 명사형. 본문 `~이다` 체.
- 문체. 두괄식 10장. 모든 슬라이드를 액션 타이틀로 종결. 한 장에 메시지 하나. 강조 색 두 개 이내.
- 분량. 8~15장. 슬라이드 한 장당 헤드라인 30자 이내, 본문 50자 이내.
- 금지. "개요", "감사합니다" 같은 정보 없는 헤드라인. 명사형 명사 나열 헤드라인("시장 동향"). 폰트와 색상의 흔들림.
- 액션 타이틀 예시. "Q3 ROAS 2.1로 목표 미달함" / "신규 채널 비중 30%로 축소 권고함" / "재무팀장 결재 후 4분기 실행함"

### 컨설팅 덱 (진단·권고)
- 종결 체. 모든 헤드라인을 액션 타이틀 `~함·~임`. 본문 `~이다` 체.
- 문체. 모든 페이지를 액션 타이틀로 종결. 한 페이지 한 메시지. So what 끝까지. 답와 일치. 강조 색 두 개 이내.
- 분량. 30~80장. 헤드라인 30자 이내, 본문 50자 이내.
- 금지. 명사형 명사 나열 헤드라인("시장 동향"). 데이터 나열 슬라이드. 권고 부재 진단. 답 불일치.

## fallback 5단계 합성 알고리즘

1. **결정자의 위치를 본다.** 임원 / 이사회 / 주주 / IC / 외부 클라이언트 중 누구인가.
2. **결정의 종류를 본다.** 승인(go/no-go) / 자원 배분 / 권고 채택 / 전략 방향 결정 중 무엇인가.
3. **두 후보를 고른다.** 결정자 일치 + 결정 종류 일치가 1순위.
4. **문체 합성.** 첫 줄에 결정 요청 또는 답 + 근거 단락 + 가정 명시. PPT 계열은 액션 타이틀 종결. 9.7 영역이면 BLACK 제약 두 가지 이상.
5. **금지 합성.** 형용사 자랑("견조한·안정적인") + 회피 어미("검토가 필요") + 약속 인플레이션 세 줄을 기본으로.

예. **"위원회 안건 보고서"**. 결정자 위원회. 결정 종류 승인. 후보 = 이사회 보고서 + 영업 제안서. 합성. 결정 요청 한 줄 + 차별성·근거 한 단락 + 일정·예산 명시. 9.7 영역이므로 자기 변호 + 모호한 "검토 중" 두 줄 직접 인용 금지.
